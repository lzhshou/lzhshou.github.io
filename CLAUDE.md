# APAAM Lab Website — Claude 项目指南

## 项目概述

APAAM Lab 官方网站，基于 Docusaurus v3.8.1 构建，React 19 + TypeScript。同时作为 Zhengshou Lai（中山大学土木工程学院副教授）的个人学术主页。

**数据来源**：所有内容来自 `myacademia/`。mywebpage 是只读展示层，不维护独立数据。

```
myacademia/                    mywebpage/
├── 00-cv/cv-zh.md    ─────→  docs/about/index.md (profile)
├── 10-publications/  ─────→  docs/publications/index.md
│   └── */*.pdf       ─────→  static/pdf/* (manual copy)
├── 12-projects/      ─────→  docs/research/projects.md
├── 22-scholars/      ─────→  docs/people/
└── 24-assets/        ─────→  static/img/
```

- **生产站点**: https://lzhshou.github.io
- **默认语言**: 中文 (`zh`)，辅语言: 英文 (`en`)
- **部署目标**: GitHub Pages (`gh-pages` 分支)

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Docusaurus v3.8.1 |
| 前端 | React 19, TypeScript |
| 样式 | CSS Modules + `src/css/custom.css` |
| 动画 | Framer Motion |
| 公式 | remark-math + rehype-katex + KaTeX CSS |
| i18n | Docusaurus 内置 i18n |
| 构建 | npm scripts + 自定义 `mywebpage` CLI |

## 核心配置

- **`docusaurus.config.ts`**: 站点元数据、导航栏、页脚、i18n、预设配置
- **`sidebars.ts`**: 各文档区的侧边栏定义（peopleSidebar, researchSidebar, publicationsSidebar, resourcesSidebar, teachingSidebar, aboutSidebar）
- **`package.json`**: 依赖管理，脚本入口

## 常用命令

```bash
# 依赖安装
npm install

# 生产构建 + 静态预览（双语言）
npm start

# 仅构建
npm run build

# 仅启动预览（需已有 build）
npm run serve

# 部署到 GitHub Pages
npm run deploy

# 热更新开发（单语言）
npx docusaurus start --locale zh
npx docusaurus start --locale en

# 类型检查
npm run typecheck

# 生成/更新翻译文件
npm run write-translations
```

### mywebpage CLI

```bash
make install                  # 安装 CLI 到 ~/.local/bin

mywebpage build               # 构建
mywebpage build -c            # 清缓存后构建
mywebpage build -s 3000       # 构建后启动服务
mywebpage deploy              # 部署
mywebpage server start        # 后台启动服务
mywebpage server stop         # 停止后台服务
mywebpage server status       # 查看状态
mywebpage server restart      # 重启
mywebpage update              # 重新安装 CLI
mywebpage uninstall           # 卸载 CLI

make uninstall                # 卸载 CLI
```

## 内容编辑规范

### 多语言同步

- 中文源文件在 `docs/`
- 英文翻译在 `i18n/en/docusaurus-plugin-content-docs/current/`
- 任何中文文档变更，需检查对应英文文件是否需要同步更新
- UI 文本在 `i18n/zh/code.json` 和 `i18n/en/code.json` 中维护

### Markdown 文件规范

- 使用标准 Markdown + MDX
- 数学公式用 `$...$`（行内）和 `$$...$$`（块级），由 KaTeX 渲染
- 图片路径：以 `/img/` 或 `/pdf/` 开头引用 `static/` 下的资源
- 内部链接使用相对路径或绝对路径（Docusaurus 会校验）

### 博客文章

- 文件名格式：`YYYY-MM-DD-title.md`
- 需要 frontmatter：`title`, `authors`, `tags`, `date`
- 作者需在 `blog/authors.yml` 中定义
- 标签需在 `blog/tags.yml` 中定义（中文），英文标签在 `i18n/en/docusaurus-plugin-content-blog/tags.yml`

### 侧边栏更新

新增文档类别时，在 `sidebars.ts` 中注册对应的 sidebar 配置。`autogenerated` 类型按目录结构自动生成，显式数组可精确控制顺序。

## 主题定制

- 全局样式：`src/css/custom.css`
- 组件覆盖：`src/theme/`（使用 Docusaurus Swizzle）
- 自定义页面：`src/pages/`（如首页 `index.tsx`）

## 部署注意事项

- GitHub Pages 部署分支为 `gh-pages`
- `npm run deploy` 需要 `GIT_USER` 环境变量（CLI 会自动尝试从 `gh` 或 git origin 推断）
- 构建时会检查 broken links，配置为 `warn` 级别不会阻断构建

## 开发约定

- Node >= 18
- 提交前运行 `npm run typecheck`
- 图片使用 kebab-case 命名（如 `cfd-dem-framework.png`）
- PDF 论文存放在 `static/pdf/first-or-correspondence/` 下
